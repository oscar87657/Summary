# Chapter 1. Java 소개 및 개발 환경

> **핵심 요약:** Java는 OOP를 대표하는 크로스플랫폼 언어로, JVM을 통해 어떤 OS에서도 동일하게 실행된다. 프로그램은 `.java` 소스파일 → 컴파일 → `.class` 바이트코드 → JVM 실행 순서로 동작한다.

## 1. Java가 왜 중요한가?

- **고수준 언어 (High-level language)**: 사람이 읽을 수 있는 코드 (`X = (Y + Z) / 2`) → 컴파일러가 바이트코드로 변환
- **언어 계층**: 바이너리(Binary) → 기계어(Machine code) → 어셈블리(Assembly) → **고수준(Java, Python)** 순으로 추상화됨
- Java의 활용 영역:
  - **데스크탑 앱**: JRE가 있는 모든 컴퓨터에서 실행
  - **서블릿(Servlet)**: 웹 서버 위에서 동작하는 Java 프로그램 (JavaScript와 다름)
  - **안드로이드(Android)**: Google 주도, Java 사용 (단, JVM 대신 Dalvik/ART 사용)

💬 **교수님 설명:** 2026년 현재 AI가 코딩을 극도로 빠르게 수행하지만, OOP 구조를 모르면 AI가 만든 수십 개의 파일을 통제할 수 없다. **풀스택 역량과 OOP 이해는 생존의 최소 요건**이다.

## 2. Java 프로그램 실행 흐름

```
소스코드 (.java)
     ↓  javac (Java 컴파일러)
바이트코드 (.class)
     ↓  JVM (Java Virtual Machine)
실행 (어떤 OS/H/W에서도 동일)
```

- **크로스플랫폼**: Write Once, Run Anywhere — JVM만 있으면 OS 무관
- **C/C++ 차이**: C/C++은 컴파일러+링커 → `.exe` 실행파일 (OS 종속적)
- **클래스 파일**: 하나의 `.java`에 여러 클래스 작성 가능, `public` 클래스는 1개만, 파일명 = public 클래스명

## 3. Java의 핵심 특징

| 특징 | 설명 |
|------|------|
| 플랫폼 독립성 | JVM을 통해 어떤 OS에서도 실행 |
| 객체지향 | 상속(Inheritance), 다형성(Polymorphism), 캡슐화(Encapsulation) |
| 가비지 컬렉션 | 메모리 해제를 JVM이 자동 처리 (개발자 부담 감소) |
| 패키지 | 강력한 표준 라이브러리 제공 |
| main() 메서드 | 프로그램 시작점, 클래스가 많아도 main()은 1개 |

## 5. Java 프로그램 기본 구성


**주석 종류:**
```java
// 한 줄 주석
```

## ⭐ 시험 포인트
- `.java` → `.class` → JVM 실행 흐름
- `public` 클래스는 파일 하나에 1개만, 파일명 = 클래스명
- Java는 GC(가비지 컬렉션)으로 메모리를 자동 해제 (`new`로 할당만 함)
- `main()` 메서드 시그니처: `public static void main(String[] args)`

# Chapter 2. 객체(Object)와 클래스(Class) 개념

> **핵심 요약:** OOP의 핵심은 현실 세계의 사물을 '객체'로 모델링하는 것이다. 클래스는 객체의 설계도이고, 객체는 클래스의 인스턴스다. 객체끼리는 메시지를 보내 상호작용한다.

## 1. 클래스(Class)와 객체(Object)

> 클래스 = **붕어빵 틀** (설계도), 객체/인스턴스 = **붕어빵** (실체)

- **객체(Object)**: 현실 세계의 사물. 유형(Account, Vehicle)도, 무형(Employee)도 가능
- **클래스(Class)**: 객체가 어떤 데이터를 가지고 어떻게 동작하는지 정의한 설계도
- **인스턴스(Instance)**: 클래스로부터 생성된 개별 객체

**UML 표기:**
```
클래스:   [Account]          (사각형, 이름만)
객체:     [<u>SV198</u>]    (사각형, 이름에 밑줄)
객체+클래스: [<u>SV198 : BankAccount</u>]
```

💬 **교수님 설명:** OOP는 현실 사고방식을 코드에 1:1로 대응시키는 패러다임이다. AI에게 시스템 구조를 지시하기에도 최적화된 방식이다.

## 2. 메시지와 메서드

- **메시지(Message)**: 객체에게 작업을 지시하는 명령
- **메서드(Method)**: 메시지를 처리하는 객체 내부의 함수
- **인수(Argument)**: 메시지와 함께 전달하는 값

```java
// 메시지 전송: 객체명.메서드명(인수);
SV198.deposit(250.00);            // SV198 계좌에 250 입금
SV198.getCurrentBalance();        // 현재 잔액 조회 (반환값 있음)
myWindow.setVisible(true);        // 창을 보이게 설정
```

- **클래스 메서드**: 클래스 자체에 정의된 메서드 (`static`)
- **인스턴스 메서드**: 개별 객체에 정의된 메서드

## 3. 데이터 값의 종류

| 종류 | 설명 | 예시 |
|------|------|------|
| **인스턴스 데이터** | 각 객체마다 개별 보유 | 각 BankAccount의 잔액(balance) |
| **클래스 데이터** | 모든 인스턴스가 공유 | BankAccount의 최소 잔액(minimum balance) |

## 4. 상속 (Inheritance)

- **정의**: 공통 특징을 상위(부모) 클래스에 정의하고, 하위(자식) 클래스가 물려받는 구조
- **슈퍼클래스(Superclass)**: 공통 특징을 가진 부모 클래스
- **서브클래스(Subclass)**: 부모로부터 상속받은 자식 클래스

```
       Account (슈퍼클래스)
      /        \
Checking      Savings  (서브클래스)
```

```
         Student
        /        \
  Graduate      Undergrad
  /   |   \    /        \
Masters Doctoral Law  Commuting Resident
```

## 5. Java 프로그램 구성 요소

**객체 선언 vs 객체 생성:**
```java
// 1단계: 선언 (이름표만 만들기, 메모리 공간 할당)
JFrame myWindow;
Customer customer;

// 2단계: 생성 (실제 객체 만들기, new 키워드)
myWindow = new JFrame();
customer = new Customer();

// 합쳐서 한 줄로:
JFrame myWindow = new JFrame();
```

> **주의**: `customer = new Customer()` 두 번 하면 첫 번째 객체의 참조가 사라진다 (메모리 낭비)

**메서드 선언 구조:**
```java
//  접근자   수식자  반환타입  메서드명  매개변수
    public  static  void    main   (String[] args) {
        // 메서드 바디 (Method Body)
    }
```

```java
import javax.swing.*;

// 메시지 출력
```

### String (문자열)
```java
String text = "Espresso";
// 인덱스:       01234567

text.substring(6, 8)   // → "so"   (6번부터 7번까지)

text.substring(1, 5)   // → "spre"
text.length()          // → 8 (문자 개수)

String str = "I Love Java and Java loves me.";

// 문자열 연결 (concatenation)
```

```java

SimpleDateFormat sdf1 = new SimpleDateFormat("MM/dd/yy");

```

💬 **교수님 설명:** 표준 클래스는 바퀴를 재발명하지 마라는 원칙. 이미 만들어진 것을 활용하는 것이 실무다.

## ⭐ 시험 포인트
- 클래스 vs 인스턴스 구분 (붕어빵 틀 vs 붕어빵)
- `선언(declaration)` vs `생성(creation)` 차이
- `substring(i, j)`: i번 인덱스부터 j-1번까지 추출
- `indexOf()`: 찾으면 첫 위치, 없으면 -1 반환
- 메서드 선언 요소: 접근자, 수식자, 반환타입, 메서드명, 매개변수

| 상속(Inheritance) | 자식 클래스가 부모 클래스의 특성을 물려받음 |
| UML | Unified Modeling Language — 객체/클래스 관계 표준 표기법 |

# Chapter 3. Java 기본 문법 I (변수, 자료형, 연산자)

> **핵심 요약:** 변수는 데이터를 담는 메모리 공간이다. Java는 강타입 언어지만 `var`로 타입 추론이 가능하며, 형 변환(Casting)을 통해 타입을 바꿀 수 있다.

## 1. 코드 구조화 원칙: Input - Compute - Output

```java
// 1. Input: 사용자로부터 데이터 받기
String input = JOptionPane.showInputDialog(null, "나이를 입력하세요:");

// 2. Compute: 데이터 처리
int age = Integer.parseInt(input);  // 문자열 → 정수 변환
int birthYear = 2026 - age;

// 3. Output: 결과 출력
JOptionPane.showMessageDialog(null, "출생연도: " + birthYear);
```

💬 **교수님 설명:** 모든 프로그램은 이 3단계 구조를 기반으로 한다. 복잡해 보여도 결국 Input → Compute → Output이다.

## 2. 변수와 상수

### 네이밍 규칙 (반드시 준수!)
| 대상 | 규칙 | 예시 |
|------|------|------|
| 클래스명 | 대문자 시작, PascalCase | `BankAccount`, `HelloWorld` |
| 변수/메서드명 | 소문자 시작, camelCase | `firstName`, `calculateTotal()` |
| 상수 | ALL_CAPS + `static final` | `MAX_SIZE`, `PI` |

```java
// 변수 (값이 바뀔 수 있음)
int score = 95;
String name = "홍길동";

// 상수 (값이 절대 안 바뀜)
static final double PI = 3.14159;
static final int MAX_SIZE = 100;
```

## 3. 기본 자료형 (Primitive Types)

| 자료형 | 크기 | 예시 | 설명 |
|--------|------|------|------|
| `int` | 4바이트 | `int x = 10;` | 정수 |
| `double` | 8바이트 | `double d = 3.14;` | 실수 (소수점) |
| `boolean` | 1비트 | `boolean b = true;` | 참/거짓 |
| `char` | 2바이트 | `char c = 'A';` | 문자 하나 |
| `long` | 8바이트 | `long l = 100L;` | 큰 정수 |
| `String` | 가변 | `String s = "hello";` | 문자열 (객체) |

### var 키워드 (타입 추론)
```java
var score = 95;         // 컴파일러가 int로 추론
var name = "홍길동";    // 컴파일러가 String으로 추론
var price = 9.99;       // 컴파일러가 double로 추론
```

💬 **교수님 설명:** `var`는 컴파일러가 데이터 타입을 추론해주므로 적극 활용을 권장한다. 타입이 명확할 때 코드가 간결해진다.

## 4. 형 변환 (Type Casting)

```java
// JOptionPane 입력값은 항상 String → 계산 시 변환 필수!
String input = JOptionPane.showInputDialog(null, "숫자 입력:");

int num = Integer.parseInt(input);      // String → int
double d = Double.parseDouble(input);   // String → double

// 숫자 간 형 변환
double result = (double) 5 / 2;   // → 2.5  (강제 변환)
int truncated = (int) 3.9;        // → 3    (소수점 버림)
```

> **주의**: JOptionPane 입력값을 바로 계산에 쓰면 문자열 연결이 됨! 반드시 parseInt 등으로 변환 후 사용

## 5. 연산자

### 산술 연산자
```java
int a = 10, b = 3;
a + b   // 13
a - b   // 7
a * b   // 30
a / b   // 3 (정수 나눗셈, 소수점 버림)
a % b   // 1 (나머지)
```

### + 연산자의 이중 역할 (오버로딩 개념)
```java
10 + 20             // → 30    (숫자끼리 → 덧셈)
"나이: " + 25       // → "나이: 25"  (문자열 포함 → 연결)
"1" + 2 + 3         // → "123"  (왼쪽부터 처리, 문자열 우선)
1 + 2 + "3"         // → "33"   (1+2=3 먼저, 그 후 "3" 연결)
```

💬 **교수님 설명:** + 연산자가 숫자 덧셈과 문자열 결합 두 가지 역할을 하는 것이 **다형성(Polymorphism)의 가장 쉬운 예시**다.

| 변수(Variable) | 값을 저장하는 메모리 공간 |
| 상수(Constant) | 한번 정해지면 바뀌지 않는 값 (`static final`) |
| 형 변환(Casting) | 데이터 타입을 다른 타입으로 변환 |
| 오버로딩(Overloading) | 같은 이름이 상황에 따라 다른 역할을 수행 |
| 타입 추론 | `var` 키워드로 컴파일러가 자동으로 타입 결정 |

# Chapter 4. Java 기본 문법 II (조건문, 반복문)

> **핵심 요약:** 조건문(if/switch)으로 흐름을 제어하고, 반복문(for/while/do-while)으로 반복 작업을 처리한다. 실무에서는 while이 가장 많이 쓰이며, 예외 처리와 함께 사용한다.

## 1. 조건문

### if - else if - else
```java
int score = 85;

if (score >= 90) {
    System.out.println("A");
} else if (score >= 80) {
    System.out.println("B");
} else if (score >= 70) {
    System.out.println("C");
} else {
    System.out.println("F");
}
```

### 삼항 연산자 (Ternary Operator)
```java
// 문법: (조건식) ? 참일 때 값 : 거짓일 때 값
int max = (a > b) ? a : b;
String grade = (score >= 60) ? "합격" : "불합격";
```

> 삼항 연산자는 단순 조건의 경우 코드를 한 줄로 간결하게 표현할 수 있다.

### switch 문
```java
int day = 3;
switch (day) {
    case 1: System.out.println("월"); break;
    case 2: System.out.println("화"); break;
    case 3: System.out.println("수"); break;
    default: System.out.println("기타"); break;
}
```

## 2. 반복문

### for 문 — 반복 횟수가 명확할 때
```java
for (int i = 0; i < 10; i++) {
    System.out.println(i);
}
```

> **교수님 강조**: 인덱스는 0부터 시작, 등호 빼기 (`i < 10`, `i < arr.length`) → Off-by-one 버그 방지

### while 문 — 조건이 참인 동안 계속
```java
while (조건식) {
    // 조건이 true인 동안 반복
}

// 실무 패턴: 프로그램이 종료 요청 전까지 계속 동작
while (true) {
    try {
        // 핵심 로직
        String input = JOptionPane.showInputDialog("입력:");
        if (input == null) break;  // 취소 시 종료
        // 처리...
    } catch (Exception e) {
        // 에러가 나도 프로그램 계속 실행
    }
}
```

💬 **교수님 설명:** 상용 소프트웨어는 유저가 끄기 전까지 계속 돌아야 하고, 잘못된 입력에도 죽지 않아야 한다. while + try-catch 조합이 이를 구현하는 기본 패턴이다.

### do-while 문 — 무조건 한 번 먼저 실행
```java
do {
    // 코드가 무조건 먼저 한 번 실행됨
    // 이후 조건 검사
} while (조건식);
```

**do-while 실무 활용:**
```java
// 예: 메뉴를 무조건 한 번 보여주고, 유효한 입력 받을 때까지 반복
String input;
do {
    input = JOptionPane.showInputDialog("1~3 중 선택:");
} while (!input.equals("1") && !input.equals("2") && !input.equals("3"));
```

💬 **교수님 설명:** do-while은 반복 전 무조건 선행 작업(초기화, 메뉴 출력 등)이 필요한 상황에서 영리하게 활용할 수 있다.

## 3. 중첩 반복문과 시간 복잡도

```java
// O(n²) — 이중 반복문
for (int i = 0; i < n; i++) {
    for (int j = 0; j < n; j++) {
        // n × n 번 실행
    }
}

// O(n) — 단일 반복문
for (int i = 0; i < n; i++) {
    // n번 실행
}
```

**Big-O 표기법 (시간 복잡도):**
| 표기 | 설명 | 비고 |
|------|------|------|
| O(1) | 상수 시간 | 배열 인덱스 접근 |
| O(log n) | 로그 시간 | 이진 탐색 |
| O(n) | 선형 시간 | 배열 순차 탐색 |
| O(n²) | 이차 시간 | 이중 반복문 |

## 4. 프로그램 실행 시간 측정

```java
Date start = new Date();
// 측정할 코드 실행
Date end = new Date();
long elapsed = end.getTime() - start.getTime();
System.out.println("실행 시간: " + elapsed + "ms");
```

## ⭐ 시험 포인트
- 반복문 3가지 차이: for(횟수 명확), while(조건), do-while(선실행 후검사)
- 삼항 연산자 문법: `(조건) ? 참 : 거짓`
- do-while: 조건이 거짓이어도 최소 1회 실행
- 인덱스는 0부터, `<` 사용 (등호 빼기)

## 📚 용어 정리
| 용어 | 설명 |
|------|------|
| 삼항 연산자 | 조건에 따라 두 값 중 하나를 선택하는 한 줄 조건문 |
| Big-O 표기법 | 알고리즘의 시간 복잡도를 나타내는 표기 |
| Off-by-one 에러 | 인덱스를 하나 더 가거나 덜 가는 흔한 버그 |

# Chapter 5. 배열 (Array)

> **핵심 요약:** 배열은 동일 타입의 데이터를 메모리에 연속적으로 배치한 자료구조다. 접근 속도는 빠르지만 크기가 고정된다. 2차원 배열은 행렬(표) 형태의 데이터 처리에 사용된다.

## 1. 배열의 본질

- **정의**: 같은 타입의 데이터를 메모리에 **연속적**으로 저장
- **장점**: 인덱스로 즉시 접근 → 빠른 탐색 (O(1))
- **단점**: 크기 고정 → 데이터 추가/삭제 불편, 메모리 낭비 가능

```
메모리:  [10][20][30][40][50]
인덱스:   0   1   2   3   4
```

💬 **교수님 설명:** 연속된 메모리 배치가 성능의 핵심이다. CPU 캐시는 연속된 메모리를 훨씬 빠르게 읽는다.

## 2. 1차원 배열

```java
// 선언 + 생성
int[] arr = new int[5];          // 크기 5의 정수 배열
String[] names = new String[3];  // 크기 3의 문자열 배열

// 선언 + 초기화
int[] scores = {90, 85, 78, 92, 88};

// 접근 및 수정
scores[0] = 100;          // 첫 번째 원소 수정
System.out.println(scores[2]);  // 세 번째 원소 출력

// 배열 순회
for (int i = 0; i < scores.length; i++) {
    System.out.println(scores[i]);
}

// for-each 순회
for (int score : scores) {
    System.out.println(score);
}
```

> **주의**: 배열 인덱스는 0부터 `length - 1`까지. `arr[arr.length]`는 ArrayIndexOutOfBoundsException!

## 3. 커스텀 메서드 (Custom Method)

```java
// 반환값 있는 메서드
static int getMax(int[] arr) {
    int max = arr[0];
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] > max) max = arr[i];
    }
    return max;
}

// 반환값 없는 메서드 (void)
static void printArray(int[] arr) {
    for (int x : arr) {
        System.out.print(x + " ");
    }
}

// 호출
int[] data = {3, 7, 1, 9, 4};
int max = getMax(data);
printArray(data);
```

**메서드의 메모리 이점 (Lazy Loading 개념):**
- 메서드 호출 시만 스택 메모리에 올라오고, 완료 후 해제
- 모든 데이터를 한 번에 메모리에 올리지 않아도 됨

💬 **교수님 설명:** 커스텀 메서드로 코드를 쪼개면 메모리 효율 + 코드 재사용성 둘 다 얻는다.

## 4. 2차원 배열

```java
// 선언 + 생성 (3행 4열)
int[][] matrix = new int[3][4];

// 선언 + 초기화
int[][] grid = {
    {1, 2, 3, 4},   // 0행
    {5, 6, 7, 8},   // 1행
    {9, 10, 11, 12} // 2행
};

// 접근
grid[1][2]  // 1행 2열 → 7

// 크기 확인
grid.length       // 행 개수 → 3
grid[0].length    // 열 개수 → 4

// 전체 순회
for (int i = 0; i < grid.length; i++) {
    for (int j = 0; j < grid[0].length; j++) {
        System.out.print(grid[i][j] + " ");
    }
    System.out.println();
}
```

💬 **교수님 강조:** 대괄호 순서는 **행-열(가로-세로)**. 앞쪽이 무조건 행(row)의 개수.

**2차원 배열 실무 활용:**
- 게임 엔진의 좌표/픽셀 처리
- 데이터 사이언스의 행렬 연산
- 탐색 방향에 맞춰 행렬을 전환(Transpose)하면 연산 속도 향상

## ⭐ 시험 포인트
- 배열 인덱스: 0 ~ `length - 1`
- `arr.length`: 배열 크기, `arr[0].length`: 2차원 배열의 열 개수
- 배열 장점: 빠른 접근(O(1)), 단점: 크기 고정
- 메서드는 호출 시에만 메모리 사용 → 효율적

## 📚 용어 정리
| 용어 | 설명 |
|------|------|
| 배열(Array) | 동일 타입 데이터의 연속된 메모리 공간 |
| 인덱스(Index) | 배열 원소의 위치 번호 (0부터 시작) |
| 레이지 로딩(Lazy Loading) | 필요할 때만 데이터를 메모리에 올리는 기술 |
| 트랜스포즈(Transpose) | 행렬의 행과 열을 바꾸는 연산 |
| 시간 복잡도 | 알고리즘 실행에 필요한 연산 횟수 |

# Chapter 6. 예외 처리 (Exception Handling)

> **핵심 요약:** 예외 처리는 프로그램이 예상치 못한 오류로 죽지 않게 방어하는 코드다. `try-catch-finally` 구조로 오류를 잡아내고 우회 코드를 실행한다.

## 1. 예외 처리가 필요한 이유

💬 **교수님 설명:** **"유저는 악마다"** — 개발자가 예상하지 못한 입력을 반드시 한다. 실무에서 80% 이상의 시간은 이런 방어 코드(예외 처리)에 투자한다.

```java
// 예외 처리 없을 때 — 잘못된 입력 시 프로그램 강제 종료
String input = JOptionPane.showInputDialog("숫자 입력:");
int num = Integer.parseInt(input);  // "abc" 입력 시 → 프로그램 죽음!
```

## 2. try-catch-finally 구조

```java
try {
    // 핵심 로직 (에러가 발생할 수 있는 코드)
    String input = JOptionPane.showInputDialog("숫자 입력:");
    int num = Integer.parseInt(input);
    System.out.println("입력값: " + num);

} catch (NumberFormatException e) {
    // 특정 에러 타입 처리
    JOptionPane.showMessageDialog(null, "숫자만 입력하세요!");

} catch (Exception e) {
    // 모든 에러 처리 (가장 넓은 그물)
    System.out.println("에러: " + e.getMessage());

} finally {
    // 에러 여부 상관없이 무조건 실행
    // 자원 해제, 연결 종료 등
    System.out.println("처리 완료");
}
```

**각 블록의 역할:**
| 블록 | 역할 |
|------|------|
| `try` | 핵심 로직. 에러가 발생하면 즉시 catch로 이동 |
| `catch` | 에러 발생 시 실행. 강제 종료 방지 |
| `finally` | 에러 유무와 관계없이 항상 실행 (자원 반환용) |

## 3. main 메서드와 String[] args

```java
public static void main(String[] args) {
    // args: 터미널에서 프로그램 실행 시 넘겨주는 인수
    // java MyProgram arg1 arg2
    System.out.println(args[0]);  // "arg1"
}
```

💬 **교수님 설명:** `String[] args`는 터미널(CLI)에서 인수를 받아 백엔드 시스템을 가동하기 위한 표준 설계다.

## 4. 에러를 두려워하지 마라

💬 **교수님 설명:** 전문가에게 에러는 catch로 잡을 힌트일 뿐이다. 에러 메시지를 읽고 원인을 찾는 능력이 실력이다. **시행착오를 통한 뇌의 성장이 유일한 실력 향상의 길**이다.

## ⭐ 시험 포인트
- `try`: 핵심 로직, `catch`: 에러 처리, `finally`: 무조건 실행
- `finally`는 자원 반환, 연결 종료 등에 활용
- 예외 처리 없으면 잘못된 입력 시 프로그램 강제 종료

## 📚 용어 정리
| 용어 | 설명 |
|------|------|
| 예외(Exception) | 프로그램 실행 중 발생하는 오류 상황 |
| try-catch | 예외를 감지하고 처리하는 문법 구조 |
| finally | 예외 발생 여부와 무관하게 항상 실행되는 블록 |
| NumberFormatException | 숫자가 아닌 값을 parseInt 등으로 변환 시 발생하는 예외 |

# Chapter 7. 프로그래머 정의 클래스 (Advanced Class)

> **핵심 요약:** 클래스는 데이터 멤버(field), 생성자(constructor), 메서드(method) 3가지로 구성된다. 정보 은닉(캡슐화)을 위해 데이터 멤버는 반드시 `private`으로 선언하고, 외부에서는 메서드를 통해서만 접근한다.

## 1. 클래스 템플릿 구조

클래스는 **설계도(template)**다. 클래스로부터 객체(인스턴스)를 만든다.

```java
// 기본 구조
public class ClassName {

    // ① 데이터 멤버 (Data Members / Fields)
    private dataType memberName;

    // ② 생성자 (Constructor) — 클래스명과 동일, 반환 타입 없음
    public ClassName(parameters) {
        // 초기화 코드
    }

    // ③ 멤버 메서드 (Member Methods)
    public returnType methodName(parameters) {
        // 메서드 바디
    }
}
```

### 메서드 선언 문법
```
<접근제어자> <반환타입> <메서드명>(<매개변수 목록>) {
    <본문>
}
```

예시:
```java
public  void  setOwnerName  (String name) { ownerName = name; }
//  ↑접근자  ↑반환타입  ↑메서드명      ↑매개변수
```

### 생성자 선언 문법
```
public <클래스명>(<매개변수 목록>) {
    <초기화 코드>
}
```

> **핵심**: 생성자는 반환 타입이 없다. 클래스명과 반드시 동일해야 한다.

## 2. Bicycle 클래스 — 완전한 예제

```java
public class Bicycle {
    // 데이터 멤버 — 모두 private!
    private String ownerName;
    private int speed;
    private int gear;

    // 생성자
    public Bicycle(String name, int speed, int gear) {
        ownerName = name;
        this.speed = speed;
        this.gear = gear;
    }

    // 메서드들
    public void setOwnerName(String name) { ownerName = name; }
    public String getOwnerName() { return ownerName; }

    public void setSpeed(int speed) { this.speed = speed; }
    public int getSpeed() { return speed; }

    public void setGear(int gear) { this.gear = gear; }
    public int getGear() { return gear; }
}
```

**사용 (main 메서드에서):**
```java
Bicycle myBike = new Bicycle("홍길동", 0, 3);
myBike.setSpeed(10);
System.out.println(myBike.getOwnerName());  // "홍길동"
System.out.println(myBike.getSpeed());      // 10
```

## 3. Account 클래스 — 입금/출금 예제

```java
public class Account {
    private String ownerName;
    private double balance;

    // 생성자
    public Account(String ownerName, double initialBalance) {
        this.ownerName = ownerName;
        balance = initialBalance;
    }

    // 메서드
    public void deposit(double amount) {
        balance += amount;
    }

    public void withdraw(double amount) {
        balance -= amount;
    }

    public double getCurrentBalance() {
        return balance;
    }

    public String getOwnerName() {
        return ownerName;
    }
}
```

**사용:**
```java
Account myAccount = new Account("홍길동", 1000.0);
myAccount.deposit(500.0);
myAccount.withdraw(200.0);
System.out.println(myAccount.getCurrentBalance());  // 1300.0
```

## 4. 정보 은닉 (Information Hiding)

> 데이터 멤버는 `private`, 메서드는 `public` — 캡슐화의 핵심 원칙

### UML 표기법
```
[Account]
- ownerName : String    ← - (마이너스) = private
- balance   : double    ← - (마이너스) = private
+ deposit(amount: double)    ← + (플러스) = public
+ withdraw(amount: double)
+ getCurrentBalance() : double
```

- `-` (minus): private 접근 제어자
- `+` (plus): public 접근 제어자

💬 **교수님 강조:** **데이터 멤버는 무조건 private!** 외부에서 직접 `myAccount.balance = -1000` 같은 조작을 막는 것이 캡슐화다. Setter를 통해 유효성 검증을 한 곳에서 관리할 수 있다.

## 5. 인수(Argument) vs 매개변수(Parameter) 구분

```java
// 메서드 선언 시 → 매개변수 (Parameter): 받는 쪽
public void setOwnerName(String name) {   // name = 매개변수
    ownerName = name;
}

// 메서드 호출 시 → 인수 (Argument): 보내는 쪽
myBike.setOwnerName("김철수");   // "김철수" = 인수
```

> **암기:** 선언할 때 = **매개변수(parameter)**, 호출할 때 = **인수(argument)**

## 6. 객체 전달 — 참조 주소(Reference) 전달

```java
// 객체를 전달할 때: 객체 복사 X, '참조 주소'만 전달
public static void upgrade(Bicycle bike) {
    bike.setGear(bike.getGear() + 1);  // 원본 객체가 변경됨!
}

Bicycle myBike = new Bicycle("홍길동", 0, 3);
upgrade(myBike);  // myBike의 gear가 4로 변경됨
System.out.println(myBike.getGear());  // 4
```

💬 **교수님 설명:** 객체를 주고받을 때 실제 데이터가 복사되는 게 아니라 메모리 주소만 전달된다. 무거운 데이터를 불필요하게 복사하는 낭비를 막는 Java의 핵심 설계다.

## 7. 지역 변수(Local Variable) vs 데이터 멤버(Field)

```java
public class Account {
    private double balance;   // ← 데이터 멤버: 객체 전체 생존 기간 유지

    public void deposit(double amount) {
        double newBalance = balance + amount;   // ← 지역 변수: 메서드 종료 시 사라짐
        balance = newBalance;
    }
}
```

- **데이터 멤버**: 클래스 전체에서 사용, 객체가 살아있는 동안 유지
- **지역 변수**: 해당 메서드(블록) 안에서만 존재, 메서드 종료 시 소멸

## 8. Dice 클래스 — 난수 활용 예제

```java
public class Dice {
    private static final int MIN_NUMBER = 1;
    private static final int MAX_NUMBER = 6;

    public int roll() {
        // Math.random() → 0.0 이상 1.0 미만 난수
        return (int) Math.floor(
            Math.random() * (MAX_NUMBER - MIN_NUMBER + 1) + MIN_NUMBER
        );
    }
}

// 사용
Dice dice = new Dice();
System.out.println(dice.roll());  // 1~6 사이 난수
```

## ⭐ 시험 포인트
- 클래스 3대 구성: **데이터 멤버**, **생성자**, **메서드**
- 데이터 멤버는 반드시 `private` → UML에서 `-` 기호
- 생성자: 클래스명과 동일, 반환 타입 없음
- **인수(argument)**: 호출할 때 전달하는 값 / **매개변수(parameter)**: 선언할 때 받는 변수
- 객체 전달 = 참조 주소만 전달 (원본 객체 변경됨)
- 지역 변수: 메서드 종료 시 소멸 / 데이터 멤버: 객체 생존 기간 동안 유지

## 📚 용어 정리
| 용어 | 설명 |
|------|------|
| 클래스 템플릿 | 객체를 찍어내는 설계도 |
| 생성자(Constructor) | 객체 생성 시 자동 호출되어 초기화하는 특수 메서드 |
| 데이터 멤버(Field) | 클래스 내부의 변수, 객체의 상태를 저장 |
| 인수(Argument) | 메서드 호출 시 전달하는 값 |
| 매개변수(Parameter) | 메서드 선언 시 받는 변수 |
| 정보 은닉 | private으로 데이터를 숨기고 메서드로만 접근하게 함 |
| UML | Unified Modeling Language — `+`=public, `-`=private |

# Chapter 8. 고급 클래스 Vol.2 — this, 오버로딩, 가비지 컬렉션, 접근 제어자

> **핵심 요약:** `this`는 현재 객체 자신을 가리키는 포인터다. `this()`는 생성자 체이닝 함수로, 둘은 전혀 다르다. 메서드 오버로딩은 매개변수 개수/타입이 다를 때만 가능하며, 반환 타입만 다른 것은 에러다. 가비지 컬렉션(GC)은 JVM이 자동으로 메모리를 회수한다.

## 1. `this` 키워드 — 자기 자신 참조

`this`는 **현재 객체 자신**을 가리키는 참조 변수다.

```java
public class Bicycle {
    private String ownerName;
    private int speed;

    // 매개변수명이 데이터 멤버명과 같을 때 → this로 구분
    public Bicycle(String ownerName, int speed) {
        this.ownerName = ownerName;  // this.ownerName = 데이터 멤버
        this.speed = speed;          // speed = 매개변수
    }

    public void setOwnerName(String ownerName) {
        this.ownerName = ownerName;  // this 없으면 매개변수에 자기 자신을 대입하는 꼴
    }
}
```

> **핵심:** `this.ownerName` = 데이터 멤버(필드), `ownerName` = 매개변수

## 2. `this()` 함수 — 생성자 체이닝 (Constructor Chaining)

> `this` 키워드와 `this()` 함수는 완전히 다른 것이다!

`this()`는 **같은 클래스의 다른 생성자를 호출**하는 함수다. 생성자의 첫 번째 줄에서만 사용 가능하다.

```java
public class Account {
    private String ownerName;
    private double balance;

    // 매개변수 있는 생성자 (주 생성자)
    public Account(String ownerName, double balance) {
        this.ownerName = ownerName;
        this.balance = balance;
    }

    // 기본 생성자 → this()로 주 생성자 호출 (코드 중복 제거!)
    public Account() {
        this("Unknown", 0.0);   // ← this() 함수: 위의 생성자를 호출
    }

    // 이름만 있는 생성자
    public Account(String ownerName) {
        this(ownerName, 0.0);   // ← this() 함수
    }
}

// 사용
Account a1 = new Account();                    // "Unknown", 0.0
Account a2 = new Account("홍길동");            // "홍길동", 0.0
Account a3 = new Account("김철수", 500000.0); // "김철수", 500000.0
```

| | `this` | `this()` |
|---|---|---|
| **종류** | 키워드 | 함수 호출 |
| **역할** | 현재 객체 자신 참조 | 같은 클래스의 다른 생성자 호출 |
| **사용 위치** | 어디서든 | 생성자의 첫 번째 줄에서만 |

## 3. 메서드 오버로딩 (Method Overloading)

**오버로딩 = 같은 이름, 다른 매개변수**

```java
public class Calculator {
    // 오버로딩 O: 매개변수 타입이 다름
    public int add(int a, int b) { return a + b; }
    public double add(double a, double b) { return a + b; }

    // 오버로딩 O: 매개변수 개수가 다름
    public int add(int a, int b, int c) { return a + b + c; }
}
```

### 오버로딩 불가 케이스 (컴파일 에러!)
```java
// 에러! 반환 타입만 다른 것은 오버로딩이 아님
public int getValue() { return 1; }
public double getValue() { return 1.0; }   // ← 컴파일 에러!
```

> **암기:** 오버로딩의 기준은 **매개변수의 개수 또는 타입**이다. 반환 타입은 오버로딩 기준이 될 수 없다.

💬 **교수님 강조:** "이름은 같지만 매개변수 개수가 다르다"는 명제를 암기할 것. 반환 타입만 다른 것은 에러다.

## 4. 객체 반환 — 참조(Reference)만 반환

```java
public class Account {
    private double balance;

    // 객체를 반환하는 메서드
    public Account createSavingsAccount() {
        Account savings = new Account();
        savings.balance = this.balance * 0.5;
        return savings;   // ← 객체 전체가 아닌 '참조 주소'만 반환
    }
}

Account checking = new Account();
Account savings = checking.createSavingsAccount();
// savings는 새로운 Account 객체의 참조 주소를 갖고 있음
```

## 5. 가비지 컬렉션 (Garbage Collection)

```java
Account a = new Account("홍길동", 1000.0);  // 객체 A 생성
a = new Account("김철수", 500.0);           // 객체 B 생성 → 객체 A의 참조가 사라짐!
// 객체 A는 이제 아무도 참조하지 않음 → JVM의 GC가 자동으로 메모리 회수
```

- **가비지(Garbage)**: 어떤 변수도 참조하지 않는 객체
- **GC(Garbage Collector)**: JVM이 자동으로 가비지를 찾아 메모리를 회수
- 개발자가 직접 `System.gc()`를 호출해 GC를 요청할 수 있으나, 실행 시점은 JVM이 결정

💬 **교수님 설명:** Java에서 `new`로 메모리를 할당만 하면 된다. C/C++처럼 `free()`나 `delete`를 수동으로 해줄 필요가 없다. JVM의 GC가 알아서 처리한다.

## 6. 4가지 접근 제어자 (Access Modifiers)

```
private  →  default  →  protected  →  public
  (좁음)                              (넓음)
```

| 제어자 | 같은 클래스 | 같은 패키지 | 자식 클래스 | 전체 |
|--------|:---------:|:---------:|:-----------:|:----:|
| `private` | O | X | X | X |
| (default) | O | O | X | X |
| `protected` | O | O | O | X |
| `public` | O | O | O | O |

```java
public class BankSystem {
    private double balance;       // 이 클래스 내부에서만
    double interestRate;          // 같은 패키지에서만 (default)
    protected String bankCode;    // 자식 클래스까지
    public String bankName;       // 어디서든
}
```

> **실무 원칙:** 데이터 멤버는 `private`, 외부에 제공할 메서드는 `public`

💬 **교수님 강조:** JavaDoc 주석으로 문서화하는 습관이 협업 수준을 높인다. 실무에서는 문서화가 필수다.

# Chapter 9. Assertion, Generic, 컬렉션 프레임워크

> **핵심 요약:** Assertion은 개발 중 논리 오류를 조기 발견한다. 제네릭은 타입을 매개변수화하여 재사용성을 극대화한다. Java Collection Framework(JCF)는 List(ArrayList/LinkedList), Map(HashMap/TreeMap) 등 표준 자료구조를 제공한다.

## 1. Assertion (단언) — 개발자용 검증 도구

```java
// 문법 1: assert 조건식;
assert balance >= 0;

// 문법 2: assert 조건식 : 메시지;
assert balance > oldBalance : "Serious Error - balance did not increase after deposit";
```

- 조건이 `true` → 정상 실행
- 조건이 `false` → `AssertionError` 발생, 프로그램 종료

### Assertion 활성화 방법
```
// 터미널에서 -ea 옵션으로 실행해야 동작함
java -ea MyProgram

// -ea 없이 실행하면 assert 구문은 완전히 무시됨
java MyProgram
```

**Assertion vs 예외 처리 비교:**
| 구분 | `try-catch` | `assert` |
|------|-------------|---------|
| 용도 | 예측 가능한 런타임 오류 방어 | 개발 중 논리적 결함 조기 발견 |
| 대상 | 유저 입력, 외부 API 등 | 내부 로직 검증 (개발자용) |
| 배포 시 | 항상 동작 | 꺼짐 (성능에 영향 없음) |

💬 **교수님 설명:** assert는 "이 조건은 절대 거짓일 수 없다"는 개발자의 선언이다. 배포 후에는 꺼지므로 성능 부담이 없다.

## 2. Java 인터페이스 (Interface)

인터페이스 = **구현 없는 설계 계약서**. 메서드 시그니처만 정의, 실제 구현은 클래스가 담당.

```java
// 인터페이스: 무엇을 할 수 있는지만 정의 (how는 없음)
public interface Drawable {
    void draw();       // 메서드 바디 없음
    int getArea();     // 메서드 바디 없음
}

// 클래스가 implements로 실제 구현 제공
public class Circle implements Drawable {
    private int radius;
    
    public void draw() { System.out.println("원 그리기"); }    // 구현 필수
    public int getArea() { return (int)(Math.PI * radius * radius); }
}
```

> 인터페이스는 데이터 멤버를 가질 수 없다 (상수만 가능). 메서드 바디가 없다.

## 3. Java Collection Framework (JCF) 전체 구조

```
java.util 패키지

Collection<E>
├── Set<E>
│   └── HashSet<E>
├── List<E>
│   ├── ArrayList<E>
│   ├── Vector<E>
│   ├── LinkedList<E>
│   └── Stack<E>
└── Queue<E>
    └── LinkedList<E>

Map<K, V>  (Collection과 별도 계층)
├── HashMap<K, V>
└── TreeMap<K, V>
```

💬 **교수님 설명:** JCF는 Java가 제공하는 표준 자료구조 모음이다. 바퀴를 재발명하지 말고 이미 최적화된 JCF를 활용하라.

## 4. List 인터페이스 — ArrayList

```java
import java.util.ArrayList;
import java.util.List;

// 선언: List 인터페이스 타입으로 받으면 다형성 활용 가능
List<String> names = new ArrayList<>();

// 핵심 메서드
names.add("홍길동");         // 끝에 추가
names.add(0, "김철수");      // 특정 인덱스에 삽입
names.get(0);                // 인덱스로 접근 → "김철수"
names.remove(0);             // 인덱스로 삭제
names.remove("홍길동");      // 값으로 삭제
names.size();                // 현재 원소 수
names.clear();               // 전체 삭제

// 순회
for (String name : names) {
    System.out.println(name);
}
```

### ArrayList vs LinkedList

| 구분 | ArrayList | LinkedList |
|------|-----------|------------|
| 내부 구조 | 동적 배열 | 이중 연결 리스트 |
| 인덱스 접근 | O(1) — 빠름 | O(n) — 느림 |
| 삽입/삭제(중간) | O(n) — 느림 | O(1) — 빠름 |
| 사용 상황 | 조회 위주 | 삽입/삭제 위주 |

## 5. Map 인터페이스 — HashMap

```java
import java.util.HashMap;
import java.util.Map;

Map<String, Integer> scores = new HashMap<>();

// 핵심 메서드
scores.put("홍길동", 95);           // 추가 (이미 있으면 덮어씌움)
scores.get("홍길동");               // 키로 값 조회 → 95
scores.containsKey("홍길동");       // 키 존재 여부 → true
scores.remove("홍길동");            // 키로 삭제
scores.size();                       // 원소 수
scores.clear();                      // 전체 삭제

// 전체 순회
for (String key : scores.keySet()) {
    System.out.println(key + ": " + scores.get(key));
}
```

### HashMap vs TreeMap

| 구분 | HashMap | TreeMap |
|------|---------|---------|
| 탐색 성능 | O(1) — 해시 | O(log n) — 트리 |
| 정렬 | 순서 없음 | 키 기준 자동 정렬 |
| 사용 상황 | 빠른 탐색 위주 | 정렬된 순서 필요 시 |

💬 **교수님 설명:** HashMap은 대용량 데이터에서 검색 속도를 수십 배 이상 단축시키는 실무의 핵심이다. **키(Key)는 절대 중복될 수 없다.**

## 6. 제네릭 (Generics) — 타입 일반화

> 제네릭 = **"도장의 인주 색깔을 고르는 것"** — 도장(구조)은 하나인데, 색깔(타입)만 바꿔 쓴다.

### 제네릭 클래스 직접 만들기

```java
// 제네릭 클래스: 타입 T를 매개변수로 받음
class Locker<T> {
    private T content;         // T 타입의 데이터 멤버

    public T retrieve() {      // T 타입 반환
        return content;
    }

    public void store(T item) { // T 타입 매개변수
        content = item;
    }
}

// 사용: T에 실제 타입을 대입
Locker<String> textLocker = new Locker<>();
textLocker.store("Hello");
String text = textLocker.retrieve();  // "Hello"

Locker<Integer> numLocker = new Locker<>();
numLocker.store(42);
int num = numLocker.retrieve();  // 42
```

**제네릭 표기 관례:**
| 표기 | 의미 |
|------|------|
| `<T>` | Type — 일반적인 타입 |
| `<E>` | Element — 컬렉션 원소 |
| `<K, V>` | Key, Value — 맵의 키와 값 |

💬 **교수님 설명:** 제네릭은 뼈대 하나로 모든 데이터 타입을 소화한다. C++의 template에 해당한다. 타입 안전성(type safety)을 보장하면서 코드 중복을 제거한다.

## 7. Vector<E> — 레거시 동적 배열

```java
import java.util.Vector;

Vector<Integer> v = new Vector<>();

v.add(10);
v.add(20);
v.add(30);

System.out.println(v.get(0));      // 10
System.out.println(v.size());      // 3
System.out.println(v.capacity());  // 내부 배열 용량 (자동 증가)
```

- ArrayList와 거의 동일하지만 **동기화(synchronized)** 지원 → 멀티스레드 환경에서 안전
- 단일 스레드 환경에서는 ArrayList가 더 빠름 (동기화 오버헤드 없음)

## 8. Iterator<E> — 순차 탐색 패턴

```java
import java.util.Iterator;
import java.util.Vector;

Vector<Integer> v = new Vector<>();
v.add(10); v.add(20); v.add(30);

// Iterator로 순회
Iterator<Integer> it = v.iterator();
while (it.hasNext()) {           // 다음 원소가 있으면
    int n = it.next();           // 다음 원소를 가져오고 커서 이동
    System.out.println(n);
}
// 출력: 10, 20, 30
```

| 메서드 | 설명 |
|--------|------|
| `iterator()` | 컬렉션에서 Iterator 객체 생성 |
| `hasNext()` | 다음 원소가 있으면 true |
| `next()` | 다음 원소를 반환하고 커서를 이동 |

## 9. 인터페이스 다형성 활용

```java
// 인터페이스 타입으로 선언 → 구현체를 바꿔도 코드 변경 최소화
List<String> list = new ArrayList<>();   // ArrayList 사용
// List<String> list = new LinkedList<>();  // 필요 시 구현체만 교체

Map<String, Integer> map = new HashMap<>();
// Map<String, Integer> map = new TreeMap<>();  // 정렬 필요 시 교체
```

💬 **교수님 설명:** 인터페이스 타입으로 변수를 선언하면 나중에 구현체를 바꾸기 쉽다. 이것이 다형성의 핵심 활용이다.

## ⭐ 시험 포인트
- `assert 조건식 : 메시지` — 조건이 false면 AssertionError, `-ea` 옵션으로 활성화
- 인터페이스: 메서드 시그니처만 정의, 구현 없음
- JCF 계층: `Collection<E>` → {`Set`, `List`, `Queue`} / `Map<K,V>`는 별도 계층
- `ArrayList` 핵심 메서드: `add()`, `get(idx)`, `remove(idx)`, `size()`, `clear()`
- `HashMap` 핵심 메서드: `put()`, `get()`, `containsKey()`, `remove()`, `size()`
- **키(Key)는 중복 불가** — 같은 키로 put하면 덮어씌워짐
- 제네릭 클래스: `class Locker<T>` — T에 실제 타입을 나중에 지정
- Iterator: `hasNext()` + `next()`로 순차 탐색
- 오버로딩 정의: **"이름은 같지만 매개변수 개수가 다르다"** (시험 단골!)
- 중간고사: 오픈북 아님! 기본 문법 + OOP 핵심 원리 숙지 필요

## 📚 용어 정리
| 용어 | 설명 |
|------|------|
| Assertion | 개발 중 논리적 결함을 조기 발견하는 검증 도구, `-ea` 옵션 필요 |
| 인터페이스(Interface) | 메서드 시그니처만 정의한 설계 계약서, 구현은 클래스가 담당 |
| JCF | Java Collection Framework — 표준 자료구조 모음 |
| ArrayList | 동적 크기 배열, 인덱스 접근 O(1) |
| LinkedList | 이중 연결 리스트, 삽입/삭제 O(1) |
| HashMap | 키-값 쌍, 해시 기반 O(1) 탐색 |
| TreeMap | 키-값 쌍, 트리 기반 O(log n), 키 자동 정렬 |
| 제네릭(Generics) | 타입을 매개변수화하여 재사용성 극대화 (`<T>`, `<E>` 등) |
| Vector<E> | 동기화 지원 동적 배열 (멀티스레드 환경) |
| Iterator | 컬렉션 원소를 순차적으로 탐색하는 객체 (`hasNext()`, `next()`) |
